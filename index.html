    How Encapsulation and Decapsulation Work

    As data travels from a sender to a receiver, it passes through networking layers. Each layer adds its own header (and sometimes a trailer) to create a Protocol Data Unit (PDU). This process is called encapsulation.

    On the way back up the stack at the receiver, each layer removes its corresponding header, restoring the original message. This reverse process is called decapsulation.

    • Application Layer — works with user data and protocols like HTTP, FTP, SMTP, and DNS.
    • Transport Layer — adds ports and reliability information (TCP/UDP), creating segments.
    • Internet Layer — adds IP addresses and routing information, forming packets.
    • Data Link Layer — adds MAC addresses and the frame check sequence (FCS), producing frames.
    • Physical Layer — converts frames into streams of bits that move across the physical medium.

    Use the controls above to step through every stage. Hover over headers and layer icons to see what each field means, and watch how the PDU grows and then shrinks again as your message successfully reaches the receiver.
